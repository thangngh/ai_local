# Gate Harness Summary

| Harness | Purpose |
| --- | --- |
| `gate` / `promote` | Runs patch command gates and promotion order from easy to extreme. |
| `noise` | Validates noisy, conflicting, and injected requirement/context paths. |
| `memory-regression` | Checks state restoration and regression-safe memory behavior. |
| `memory-layers` | Covers memory levels, evidence, scope, and confirmation. |
| `composite` | Bridges task, patch, evidence, score, risk, and decision modules. |
| `decision` | Exercises accept, retry, ask, stop, and conflict decisions. |
| `retrieval` | Validates noisy bilingual retrieval and decision coupling. |
| `agent-loop` | Covers the core state machine and deep agent loop flow. |
| `big-harness` | Validates global run limits and mandatory top-level gates. |
| `small-patch` | Constrains focused patch evidence, scope, and checks. |
| `patch-pipeline` | Covers objective-to-diff-to-test-to-evaluation flow. |
| `patch-levels` | Defines easy through extreme patch limits and risk ceilings. |
| `evaluation` | Scores correctness, evidence, risk, tests, and requirements. |
| `confirmation` | Tests confirmation branches from simple ambiguity to deep risk. |
| `knowledge` | Covers knowledge levels, verification, conflict, and policy. |
| `evidence-rank` | Ranks source authority, freshness, relevance, and penalties. |
| `tool-combo` | Checks custom tools with memory and cross-module decisions. |
| `skills` | Validates custom skill loading and module integration. |
| `memory-sql` | Covers memory SQL schema and deep memory conflicts. |
| `conflict-paths` | Exercises forced-path, no-path, and rollback conflicts. |
| `multi-conflict` | Handles equal-authority multi-instance contradictions. |
| `request-lifecycle` | Connects gateway through final answer, retry, ask, or stop. |
| `prompt-injection` | Detects injection and selects polite refusal/deny behavior. |
| `thread-control` | Covers pause, resume, interrupt, archive, and thread conflicts. |
| `operational-safety` | Covers queue, worker, outbox, audit, and prompt firewall recovery. |
| `memory-governance` | Governs write/retrieval scores, confirmation, demotion, and archive. |
| `flow-memory-rating` | Covers flow match, role binding, interference, and token pruning. |
| `global-developer` | Validates developer phase coverage for functional and non-functional items. |
